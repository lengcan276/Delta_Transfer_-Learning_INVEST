# Phase 3.6 Step 4.3 — Edits with dependencies on Phase 4 (raw-output provenance)

The following recommended edits or annotations **should NOT be finalized**
until Phase 4 (raw ricc2 / control / slurm provenance) has verified the
underlying numbers and labels. If Phase 4 finds problems, these edits
will need to be revised.

---

## D-1. Per-molecule SCS-CC2 ΔE_ST numeric values

Affects:
- `figures/caption_data/Fig4_crosscheck.json` line 86 (range "10.1–193.7 meV", mean "109.8 meV")
- `figures/caption_data/Fig4_crosscheck.json` line 90 (range "10–194 meV", mean "110 meV")
- Any wording in main.tex / Tables that quotes specific cross-method shifts
- `scs_cc2_extended_n13.per_molecule` per-mol values in canonical_metrics.json

**Why dependent:** These ranges come from
`results/scscc2_extension_n13/cross_check_n13.csv`, which is in turn
generated by parsing `ricc2_scscc2_sing.out` / `ricc2_scscc2_trip.out`
files via `scripts/scscc2_extension/parsers/parse_scscc2_dest.py`.

Phase 4 will independently re-parse the raw ricc2 outputs and confirm
(a) the files exist locally, (b) they show "ricc2 ended normally", (c)
the parsed S1/T1 values match those in `cross_check_n13.csv`, and (d)
the `control` files declare the SCS-CC2 method and def2-SVP basis as
claimed.

If Phase 4 finds any (mol, S1, T1) mismatch, the 10.1–193.7 meV range
and 109.8 meV mean will need to be recomputed.

**Direction-of-change is safe:** the wording shift from n=4 / 100–174
meV to n=13 / 10–194 meV is correct regardless of whether Phase 4 finds
specific value mismatches.

## D-2. Hz_POZ1_NPh21_CF31 — basis / method labeling

Affects:
- The Phase 3.6 recommended Table 1 caption naming Hz_POZ1_NPh21_CF31
  as the borderline-promoted molecule
- Any Methods-section wording about whether the SCS-CC2 calc for this
  molecule actually used def2-SVP (vs def2-TZVP, which has been
  discussed elsewhere in the project)
- Recommended `borderline_low_evidence_warning` annotation
- Recommended Phase 5 explicit basis-set sensitivity check on this molecule

**Why dependent:** `validated_candidates_master.csv` lists this
molecule with method label "ADC(2)/ADC(2) + SCS-CC2/TZVP" in some rows
and "SCS-CC2/def2-SVP" elsewhere. Phase 4 must read the actual control
file for this molecule's SCS-CC2 calc and confirm the basis.

If Phase 4 finds the SCS-CC2 calc actually used def2-TZVP (not
def2-SVP as the paper currently claims), every mention of "SCS-CC2/
def2-SVP" for this molecule in the manuscript will need correction —
not a wording change but a fact change.

## D-4. "Bit-identical precision" wording

Affects:
- `paper/main.tex` line 74: "Phase 1 sanity, the new pipeline recovers the historical SCS-CC2 ΔE_ST = −220.33 meV to all 7 printed decimals"
- `paper/main.tex` line 150: "Phase 1 sanity, where the new pipeline recovers the historical SCS-CC2 ΔE_ST = −220.33 meV to bit-identical precision"
- `paper_overleaf/main.tex` lines 74, 150 (mirrors)

**Why dependent:** The claim "bit-identical" is meaningful only if
verified against the historical `ricc2_scscc2_sing.out` and
`ricc2_scscc2_trip.out` files for Hz_DMAC1_NPh21_CF31. If those raw
files are missing or differ, the wording must downgrade to e.g. "with
parser-recovered ΔE_ST agreeing to printed precision" or "within 0.1
meV".

Phase 4 must:
1. Locate the historical files for Hz_DMAC1_NPh21_CF31.
2. Locate the Phase-1 sanity files for Hz_DMAC1_NPh21_CF31.
3. Run the parser on both, compare numerically.
4. Optionally also compute file-level sha256 if claim is "bit-identical
   at file level" (it is currently claimed only at the parsed-value
   level — "to all 7 printed decimals").

If the comparison is parsed-value identical but file-level different
(possible — different timestamps, different node hostnames in the
output header), the wording stays valid but could be tightened.

## D-5. Any claim that includes "real Turbomole calculation"

Affects:
- General trust in the n=13 cohort

**Why dependent:** Phase 4's whole point is to confirm that the
`ricc2_scscc2_*.out` files in `results/scscc2_extension_n13/Hz_*/turbo_*/`
are genuine Turbomole output rather than fabricated text. The patch
generated in Phase 3.6 does not make any such claim; it only rewrites
the cohort size and statistical interpretation.

If Phase 4 finds fabricated or missing raw files for any cohort member,
the entire n=13 claim collapses (or shrinks to n < 13). The recommended
patch should then be revised accordingly before any apply.
